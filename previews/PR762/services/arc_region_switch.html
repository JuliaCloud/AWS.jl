Main.Arc_Region_Switch.approve_plan_execution_step — Function
approve_plan_execution_step(approval, execution_id, plan_arn, step_name)
approve_plan_execution_step(approval, execution_id, plan_arn, step_name, params::Dict{String,<:Any})

Approves a step in a plan execution that requires manual approval. When you create a plan, you can include approval steps that require manual intervention before the execution can proceed. This operation allows you to provide that approval.

You must specify the plan ARN, execution ID, step name, and approval status. You can also provide an optional comment explaining the approval decision.

Arguments

  • approval: The status of approval for a plan execution step.
  • execution_id: The execution identifier of a plan execution.
  • plan_arn: The Amazon Resource Name (ARN) of the plan.
  • step_name: The name of a step in a plan execution.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "comment": A comment that you can enter about a plan execution.

Main.Arc_Region_Switch.create_plan — Function
create_plan(execution_role, name, recovery_approach, regions, workflows)
create_plan(execution_role, name, recovery_approach, regions, workflows, params::Dict{String,<:Any})

Creates a new Region switch plan. A plan defines the steps required to shift traffic from one Amazon Web Services Region to another.

You must specify a name for the plan, the primary Region, and at least one additional Region. You can also provide a description, execution role, recovery time objective, associated alarms, triggers, and workflows that define the steps to execute during a Region switch.

Arguments

  • execution_role: An execution role is a way to categorize a Region switch plan.
  • name: The name of a Region switch plan.
  • recovery_approach: The recovery approach for a Region switch plan, which can be active/active (activeActive) or active/passive (activePassive).
  • regions: An array that specifies the Amazon Web Services Regions for a Region switch plan. Specify two Regions.
  • workflows: An array of workflows included in a Region switch plan.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "associatedAlarms": The alarms associated with a Region switch plan.
  • "description": The description of a Region switch plan.
  • "primaryRegion": The primary Amazon Web Services Region for the application. This is the Region where the application normally runs before any Region switch occurs.
  • "recoveryTimeObjectiveMinutes": Optionally, you can specify an recovery time objective for a Region switch plan, in minutes.
  • "reportConfiguration":
  • "tags": The tags to apply to the Region switch plan.
  • "triggers": The triggers associated with a Region switch plan.

Main.Arc_Region_Switch.start_plan_execution — Function
start_plan_execution(action, plan_arn, target_region)
start_plan_execution(action, plan_arn, target_region, params::Dict{String,<:Any})

Starts the execution of a Region switch plan. You can execute a plan in either graceful or ungraceful mode.

Specifing ungraceful mode either changes the behavior of the execution blocks in a workflow or skips specific execution blocks.

Arguments

  • action: The action to perform. Valid values are activate (to shift traffic to the target Region) or deactivate (to shift traffic away from the target Region).
  • plan_arn: The Amazon Resource Name (ARN) of the plan to execute.
  • target_region: The Amazon Web Services Region to target with this execution. This is the Region that traffic will be shifted to or from, depending on the action.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "comment": An optional comment explaining why the plan execution is being started.
  • "latestVersion": A boolean value indicating whether to use the latest version of the plan. If set to false, you must specify a specific version.
  • "mode": The plan execution mode. Valid values are graceful, for starting the execution in graceful mode, or ungraceful, for starting the execution in ungraceful mode.
  • "recoveryExecutionId": The execution identifier of the recovery execution that ran in the opposite region post-recovery is ran in. Required when starting a post-recovery execution.
