Main.Nova_Act.create_act — Function
create_act(session_id, task, workflow_definition_name, workflow_run_id)
create_act(session_id, task, workflow_definition_name, workflow_run_id, params::Dict{String,<:Any})

Creates a new AI task (act) within a session that can interact with tools and perform specific actions.

Arguments

  • session_id: The unique identifier of the session to create the act in.
  • task: The task description that defines what the act should accomplish.
  • workflow_definition_name: The name of the workflow definition containing the session.
  • workflow_run_id: The unique identifier of the workflow run containing the session.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "clientToken": A unique, case-sensitive identifier that you provide to ensure the idempotency of the request.
  • "toolSpecs": A list of tool specifications that the act can invoke to complete its task.

Main.Nova_Act.create_workflow_definition — Function
create_workflow_definition(name)
create_workflow_definition(name, params::Dict{String,<:Any})

Creates a new workflow definition template that can be used to execute multiple workflow runs.

Arguments

  • name: The name of the workflow definition. Must be unique within your account and region.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "clientToken": A unique, case-sensitive identifier that you provide to ensure the idempotency of the request.
  • "description": An optional description of the workflow definition's purpose and functionality.
  • "exportConfig": Configuration for exporting workflow execution data to Amazon Simple Storage Service.

list_acts(workflow_definition_name, params::Dict{String,<:Any})

Lists all acts within a specific session with their current status and execution details.

Optional Parameters

Optional parameters can be passed as a params::Dict{String,<:Any}. Valid keys are:

  • "maxResults": The maximum number of acts to return in a single response.
  • "nextToken": The token for retrieving the next page of results.
  • "sessionId": The unique identifier of the session to list acts for.
  • "sortOrder": The sort order for the returned acts (ascending or descending).
  • "workflowRunId": The unique identifier of the workflow run containing the session.
